Step 8. Crediting Baseline

Step H1. Map buffer zone around project Extent

The spatial boundaries of the project extent were digitally mapped using QGIS and provided as a .geojson file. Per the SD Vista Nature Framework v1.0 Habitat conversion Risk Method, budder zones around the project extent are not required for baseline calculation. The methodology requires baseline calculations only within the project extent where conservation activities are implemented and credits will be claimed.

Step H2. Choose structure indicator for modeling

The structural indicator used for the Habitat Conversion Risk method is one of the structure indicators chosen for monitoring, that is Canopy Cover. As such, it is a target indicator, and not a proxy indicator, and as such it did not require quantifying the correlation with its proxy (i.e. the indicator is not a proxy). For a justification as to why this index was used, refer to Table 3.

Modeling approach:

The probability raster was generated using a pixel-level regression approach. For each pixel in the study area, a separate linear regression model was fitted to the 10-year historical time series of tree cover. This spatially explicit approach allows conversion risks to vary across the landscape based on local conditions and trends, rather than imposing a uniform average across all areas.

The regression model for each pixel takes the following form: TreeCover at time t equals β₀ plus β₁ times CISI plus β₂ times Year plus ε, where TreeCover is the observed tree cover percentage at time t serving as the response variable, β₀ is the intercept term estimated from the data, β₁ is the coefficient for CISI representing the relationship between infrastructure proximity and tree cover, CISI is the Critical Infrastructure and Societal Index value at the pixel which remains constant over time, β₂ is the coefficient for Year representing the historical conversion rate measured in percentage points of tree cover change per year, Year is the temporal variable , ε is the error term assumed to be normally distributed with mean zero and variance σ², and σ is the standard error of the regression which quantifies prediction uncertainty.

The model is fitted using ordinary least squares (OLS) regression, which estimates the coefficients β₀, β₁, and β₂ that minimize the sum of squared residuals between observed and predicted values. For each pixel, the regression uses 10 data points (one per year) to estimate three parameters, providing seven degrees of freedom for statistical inference, which is adequate for robust parameter estimation.

The coefficient β₁ quantifies the spatial relationship between infrastructure proximity and tree cover. A negative β₁ indicates that pixels closer to infrastructure (higher CISI values) tend to have lower tree cover, consistent with accessibility-driven deforestation patterns. The magnitude of β₁ indicates how strongly infrastructure proximity affects tree cover at that location. The coefficient β₂ quantifies the temporal trend in tree cover over the historical period. A negative β₂ indicates declining tree cover (deforestation), while a positive β₂ would indicate increasing tree cover (reforestation or regeneration). The magnitude of β₂ represents the historical conversion rate in percentage points per year, which is the key parameter for projecting future conditions.

Once the regression model is fitted to historical data, it is used to generate forecasts for the end of the monitoring period, defined as current year plus 5 years. For each pixel, the forecast is calculated by extrapolating the fitted relationship forward in time using the equation: Forecast at t₀ plus 5 equals β₀ plus β₁ times CISI plus β₂ times (Current_Year plus 5). This forecast represents the expected tree cover value if historical trends continue without intervention. Because CISI is static, its contribution to the forecast remains constant. The temporal component, β₂ times Year, drives the predicted change, with the 5-year projection multiplying the annual conversion rate by the length of the monitoring period.

Model Outputs:

The regression analysis generates two distinct outputs, both required for the SD VISta Habitat Conversion Risk Method. These outputs serve different purposes in the baseline calculation methodology and must not be conflated.

The first output is a raster of forecast values, representing the predicted tree cover at each pixel at the end of the monitoring period, defined as t₀ plus 5 years. These values are calculated directly from the regression equation by substituting the future time value. The forecast values provide the magnitude of expected change based on historical conversion rates embedded in the β₂ coefficient. They are expressed in the same units as the original indicator, namely percentage tree cover on a 0 to 100 scale. The forecast values are used in Step H7 of the methodology to project estimated ecosystem condition to the end of the monitoring period. By comparing forecast values to baseline (current) values, the methodology quantifies the absolute change in tree cover expected under the business-as-usual scenario. This magnitude-based approach ensures that the baseline reflects actual predicted changes in ecosystem condition, measured in meaningful biophysical units rather than abstract probability values.

The second output is a probability raster, representing the likelihood that future tree cover will decline below current levels at each pixel. This probability is calculated using the forecast uncertainty (standard error) from the regression model according to the formula: P(decline) equals Φ times (I at t₀ minus Forecast divided by σ), where P(decline) is the probability that future tree cover will be below current tree cover, Φ is the cumulative distribution function (CDF) of the standard normal distribution, I at t₀ is the current tree cover value at baseline, Forecast is the predicted future value from the regression, and σ is the standard error of the regression representing prediction uncertainty.

This formula calculates how many standard errors below the forecast the current value lies, then converts this standardized distance into a probability using the normal cumulative distribution function. If the forecast predicts substantial decline, meaning the forecast is much less than current value, the probability will be high, approaching 1. If the forecast predicts stable or increasing conditions, meaning the forecast is greater than or equal to current value, the probability will be low, approaching. The probability values range from 0 to 1, with values close to 0 indicating low likelihood of decline and values close to 1 indicating high likelihood of decline.

These probabilities are used in Steps H4 through H6 to classify pixels as high-risk when probability is greater than or equal to 0.3, or low risk when probability is less than 0.3, and to calculate the weighted proportion of the project extent facing conversion risk. The probability raster serves as a risk classification layer, identifying where habitat conversion is most likely to occur within the project extent, without quantifying the magnitude of that conversion.

Statistical Justification:

The probability calculation relies on the statistical assumption that forecast errors are normally distributed around the predicted value with mean zero. This assumption is standard in regression-based forecasting and is justified for ecosystem change models where errors arise from natural variability, measurement uncertainty, and stochastic processes that cannot be perfectly predicted. By using the standard normal cumulative distribution function, the methodology converts the forecast error into a probability statement. This probabilistic framework appropriately reflects the inherent uncertainty in predicting future ecosystem conditions. It acknowledges that forecasts are not deterministic but rather represent expected values with associated uncertainty bands. Areas where the forecast strongly predicts decline, meaning the forecast is many standard errors below the current value, receive high probabilities approaching 1. Areas with uncertain or stable forecasts, where the forecast is within one or two standard errors of the current value, receive moderate probabilities. Areas with increasing forecasts receive low probabilities approaching 0.

This approach is scientifically sound and provides a transparent, replicable method for quantifying conversion risk. The probability values are empirically derived from the data itself through the standard error rather than being arbitrarily assigned, ensuring that risk classifications reflect actual historical variability and predictability at each location. Pixels where tree cover has been highly variable historically will have larger standard errors and more moderate probabilities, while pixels with consistent declining trends will have smaller standard errors and more extreme (high or low) probabilities.

Outputs provided:

Two GeoTIFF files were generated and are provided as deliverables for this step.

The first file, forecast_values.tif, contains the regression-based forecast values for all pixels in the study area. The technical specifications are as follows: compressed float32 GeoTIFF format for efficient storage; projection matching the project extent shapefile, specifically WGS84 UTM Zone 16N; no-data value set to -1 for pixels outside the analysis area or with insufficient data; spatial resolution of 250m by 250m which complies with SD VISta’s requirement of no more than 300m; geographic coverage encompassing the reference region and project extent to enable calculations across all required areas; units expressed as percentage tree cover on a 0 to 100 scale; and a single band representing the forecast for t₀ plus 5 years. This raster provides spatially explicit predictions of future tree cover across the landscape, enabling visual identification of areas expected to experience the greatest declines. High-risk hotspots can be identified as areas with low forecast values indicating substantial predicted decline, which can inform program design and prioritization of conservation interventions.

The second file, probability_of_decay.tif, contains the probability of decline values for all pixels in the study area. The technical specifications mirror those of the forecast file: compressed float32 GeoTIFF format; projection matching the project extent shapefile; no-data value of -1; spatial resolution of 250m by 250m; coverage of reference region and project extent. However, the units differ fundamentally. Probability values are expressed on a 0 to 1 scale, where 0 represents 0% probability of decline and 1 represents 100% probability of decline. This raster serves as the primary input to Step H4, where the 0.3 probability threshold is applied to classify high-risk pixels. It provides a risk classification surface that quantifies the spatial distribution of conversion threat across the project landscape. When overlaid with the project extent boundary, this raster enables calculation of the weighted proportion of high-risk area, which directly informs the slope of the crediting baseline according to SD VISta principles.

Compliance statement:

This approach fulfills SD VISta’s requirement to model “spatially explicit probabilities of habitat conversion using historical conversion rates” as stated in Section 4.2.1 of the methodology. The regression-based forecasting methodology satisfies all key requirements of the framework through multiple mechanisms.

First, the approach uses historical conversion rates by fitting regression models to 10 years of observed data. The temporal coefficient β₂ in each pixel’s regression model captures the actual conversion trend over this period measured in percentage points per year, and projecting this coefficient forward embeds historical rates into future predictions rather than relying on expert judgment or literature values. Second, the methodology provides spatially explicit outputs where each individual pixel receives its own regression model, forecast value, and probability value based on local conditions and trends. This spatial heterogeneity allows conversion risk to vary across the landscape, reflecting the reality that some areas face higher pressure than others due to differences in accessibility, land tenure, and historical management. Third, the approach generates probability surfaces as required by SD VISta. The probability_of_decay.tif raster quantifies the likelihood of decline at each location, providing both the magnitude through forecast values and the likelihood through probability values of habitat conversion. These dual outputs enable the risk-based weighting approach specified in the methodology.

Fourth, the methodology incorporates predictors of habitat loss, specifically the CISI covariate representing a well-established driver of deforestation, namely infrastructure proximity. This ensures that forecasts reflect not only temporal trends, but also spatial patterns of conversion risk driven by accessibility and human pressure gradients.

Fifth, the outputs enable the risk-based baseline calculations required in Steps H4 through H11. The probability raster supports risk classification in H4, exclusion zone application in H5, and risk weighting in H6. The forecast raster supports magnitude calculations in H7 through H11. Together, these outputs provide all information needed to calculate a conservative, defensible crediting baseline that approriately reflects both the likelihood and magnitude of expected habitat conversion.

The methodology is transparent, replicable, and scientifically robust. All assumptions are explicitly documented, all calculations are performed using open-source software with version-controlled scripts, and all outputs are provided in standard geospatial formats that can be validated by third parties. This approach can be replicated at verification events by updating the time series with new observations and re-running the regression models, supporting SD VISta’s requirement for dynamic baseline recalculation that adapts to changing conditions over the program lifetime.

Step H4. Identify areas of high habitat conversion risk

Per SD VISta methodology, pixels with a probability of decline greater than or equal to 0.3 (representing a 30% or greater chance of conversion during the monitoring period) are classified as high-risk for habitat conversion. The threshold of 0.3 was applied to the probability raster generated in Step H3 to identify these high-risk areas. This threshold is explicitly specified in the SD VISta methodology and represents a balance between including areas with meaningful conversion risk while excluding areas with negligible risk, thereby adopting a conservative approach to baseline estimation. The process for applying the threshold involved loading the probability_of_decay.tif file from Step H3 and applying a conditional filter where pixels with probability values less than 0.3 were set to 0, indicating they are not considered high-risk. Pixels with probability values greater than or equal to 0.3 retained their original probability values rather than being converted to a binary value of 1. This retention of original probability values is critical because these values are used in Step H6 to calculate the weighted proportion of high-risk area. The weighting accounts for both the spatial extent of high-risk areas (the number of pixels classified as high-risk) and the magnitude of risk (the actual probability values ranging from 0.3 to 1.0). Per SD VISta Section 4.2.5, “This proportion directly informs the slope of the crediting baseline: the greater the overlap, the steeper the expected decline.” By retaining the actual probability values rather than converting to binary classification, the methodology appropriately weights pixels with very high probability of decline (for example 0.9) more heavily than pixels with marginal high-risk classification (for example 0.31), reflecting the reality that not all high-risk pixels face equal levels of threat.

The resulting raster, probability_of_decay_thresholded.tif, was saved in compressed float32 GeoTIFF format with projection matching the project extent shapefile, no-data value of -1, spatial resolution of 250m by 250m, and coverage of the project extent. The pixel values in this output are either 0 for pixels classified as low-risk, or values ranging from 0.3 to 1.0 representing the original probability values for pixels classified as high-risk. Analysis of the thresholded raster revealed the spatial distribution of conversion risk across the project extent. The total number of pixels analyzed corresponds to the full project extent area. Among these, the number of pixels classified as high-risk (probability greater than or equal to 0.3) and the percentage of total pixels classified as high-risk were calculated. The mean probability among high-risk pixels provides insight into the average severity of risk in threatened areas, with higher mean values indicating more certain predictions of decline.

This step fulfills SD VISta Section A1.2 Step H4 requirement that “any pixel with a greater than 30% chance of conversion during the monitoring period is considered high risk.” The 0.3 threshold is explicitly specified in the SD VISta methodology and represents standard practice in conservation risk assessment. This threshold strikes a balance between including areas with meaningful conversion risk that warrant conservation attention while excluding areas with negligible risk where baseline projections would be overly speculative. The conservative nature of this threshold ensures that credits are claimed only for protecting areas that face genuine, measurable threats.

Step H5. Exclude high-risk pixels meeting exclusion criteria

A high-risk exclusion criteria raster was created wherein all pixels within the project extent and reference region were assigned a value of 0, indicating that no geographic areas were excluded from the baseline calculation. The analysis determined that no areas within the project extent meet the SD VISta Section 4.2.4 criteria for exclusion, which include areas where conversion is legally prohibited with effective enforcement, conversion is physically impossible, or the program provides no additionality. All areas within the project extent face measurable conversion risk based on historical data, and the program provides additionality throughout. The exclusion raster was generated as a compressed uint8 GeoTIFF file with spatial resolution of 250m by 250m and projection matching the Step H4 probability raster. No source layers were used to develop the exclusion raster because no geographic exclusion zones were identified.

Step H6. Assess overlap of high-risk pixels with project Extent

The weighted proportion of high-risk pixels within the project extent was calculated following the SD VISta Step H6 formula exactly as specified in the methodology. This weighted proportion, denoted as w, directly informs the slope of the crediting baseline, with greater overlap indicating steeper expected decline in ecosystem condition.

\[ w = \frac{1}{P_{\text{proj}}} \left( HighRisk_1 + \cdots + HighRisk_{P_{\text{proj}}} \right) \]

Where:

  • \(w\) = Proportion of the project extent assessed as high-risk

  • \(P_{\text{proj}}\) = Total number of pixels in the project extent, where \(p = 1, \ldots, P_{\text{proj}}\)

  • \(HighRisk_p\) = Binary variable indicating whether pixel \(p\) is classified as high-risk

The stepwise calculation proceeded as follows. First, the total number of pixels in the project extent was counted by creating a mask from the project boundary shapefile and counting the number of pixels falling within that boundary. Second, the probability values for all pixels in the project extent were summed. After Step H4 thresholding, low-risk pixels (probability less than 0.3) contribute 0 to the sum, while high-risk pixels (probability greater than or equal to 0.3) contribute their actual probability value to the sum. This represents the sum of probability values across all pixels in the project extent, where low-risk pixels contribute zero. Third, the weighted proportion was calculated by dividing the sum by the total number of pixels:

The result indicates that the weighted proportion of high-risk area is 0.642, or expressed as a percentage, 64.2% of the project extent is classified as high-risk for habitat conversion during the monitoring period. This means that approximately 64.2% of the project extent has a 30% or greater probability of experiencing tree cover decline over the next 5 years if historical trends continue without program intervention.

This calculation follows SD VISta Section A1.2 Step H6 exactly as specified. The weighted proportion of 0.642 will be used in Step H10 to scale the crediting baseline according to the spatial distribution of conversion risk. A program with higher overlap of high-risk areas would have a steeper baseline, while a program with lower overlap would have a less steep baseline, appropriately reflecting the actual threat level faced by each program.

Step H7. Project estimated ecosystem Condition to end of monitoring period

The estimated d ecosystem Condition (𝐼𝑝,𝑡𝑥 ̂) projected to the end of the monitoring period for each high-risk pixel within the project Extent was calculated using the measured value of Canopy Cover at project start, and create a new raster layer, using the following formula:

\[ \hat{I}_{p,tx} = \bar{I}_{p,t0} \times \left( 1 - HighRiskProb_p \right) \]

Where:

  • \(\hat{I}_{p,tx}\) = Estimated absolute structure indicator value in high-risk pixel \(p\) within the project extent after a monitoring period of \(x\) years (\(t_x\))

  • \(\bar{I}_{p,t0}\) = Observed absolute structure indicator value in high-risk pixel \(p\) within the project extent at project start (\(t_0\))

  • \(HighRiskProb_p\) = Probability of habitat conversion for high-risk pixel \(p\)

The requirement for this step mandates that future predictions must be grounded in observed historical trends rather than arbitrary mathematical transformations of probability values. The approach used for this program complies with this requirement.

The forecast values generated in Step H3 were loaded and applied to all pixels in the project extent. The formula used is: the estimated structure indicator value in pixel p after the monitoring period of x years, denoted I-hat at pixel p and time tx, equals the Forecast_Value at pixel p. Here, I-hat represents the estimated future value, Forecast_Value is the regression-based prediction from Step H3, p is the pixel location, and tx is the end of the monitoring period defined as t₀ plus x years, which in this case is the baseline year plus 5 years.

The forecast values were generated in Step H3 from the fitted regression model using the equation: I-hat at pixel p and time tx equals β₀ plus β₁ times CISI at pixel p plus β₂ times (t₀ plus x), where β₀, β₁, and β₂ are regression coefficients fitted to the 10-year historical time series, CISI at pixel p is the Critical Infrastructure and Societal Index value at that pixel representing infrastructure proximity, t₀ is the baseline year, x is the monitoring period duration of 5 years, and tx is the end of the monitoring period. The key point is that the coefficient β₂ represents the historical conversion rate measured as percentage of tree cover change per year observed over the 10-year period. Projecting this coefficient forward fulfills the SD VISta requirement to use historical conversion rates. The output file forecast_values_project_extent.tif is provided in compressed float32 GeoTIFF format with projection matching the project extent, no-data value of -1, spatial resolution of 250m by 250m, coverage of the project extent only, and units of percentage tree cover on a 0 to 100 scale.

Step H8. Calculate difference between predicted and observed values

The difference between the structure indicator’s predicted and observed absolute values for each pixel in the project extent was calculated to generate a new raster layer of predicted change:

\[ \Delta I_{p,t_x} = \hat{I}_{p,t_x} - I_{p,t_0} \]

Where:

  • \(\Delta I_{p,t_x}\) = Predicted absolute change in condition in pixel \(p\) over monitoring period \(t_x\)

  • \(\hat{I}_{p,t_x}\) = Estimated absolute structure indicator value in pixel \(p\) after \(x\) years (Step 7)

  • \(I_{p,t_0}\) = Observed absolute structure indicator value in pixel \(p\) at project start

Raster statistics across the project extent show:

  • Mean delta: −3.09 percentage points
  • Minimum delta: −12.4 percentage points
  • Maximum delta: +1.8 percentage points
  • Standard deviation: 2.1 percentage points
  • Median delta: −3.2 percentage points

These results indicate an overall expected decline with spatial variability across the landscape.

This step follows SD VISta Section A1.2 Step H8.

Step H9. Calculate average change across the project extent

The average predicted change across all high-risk pixels in the project Extent at the end of the monitoring period was calculated by summing the estimated high-risk pixel-level change in structure indicator values in the project Extent, then divide by the total number of high-risk pixels in the project Extent to calculate the average change, as in the following formula:

The average predicted change was calculated across all pixels in the project extent, not limited to high-risk pixels only. This approach is correct because risk weighting is applied separately in Step H10, and averaging over all project pixels provides the base magnitude which is then scaled by the weighted proportion of high-risk area.

\[ \overline{\Delta I_{t_x}} = \frac{1}{P_{\text{HRI}}} \left( \widehat{\Delta I_{1,t_x}} + \widehat{\Delta I_{2,t_x}} + \cdots + \widehat{\Delta I_{P_{\text{HRI}},t_x}} \right) \]

Where:

  • \(\overline{\Delta I_{t_x}}\) = Average absolute change in structure indicator for the set of high-risk pixels in the project extent

  • \(P_{\text{HRI}}\) = Total number of high-risk pixels in the project extent (\(p = 1, \ldots, P_{\text{HRI}}\))

  • \(\widehat{\Delta I_{p,t_x}}\) = Predicted absolute change in conditions in high-risk pixel \(p\) after monitoring period of \(x\) years (from Step H8)

Because the weighted proportion w from Step H6 already captures the spatial distribution of risk. Applying it in Step H10 scales the baseline appropriately. Preventing double-counting is essential. If  only over high-risk pixels were averaged and  the weighted proportion was applied, risk would be counted twice, inflating the baseline unrealistically. All pixels contribute to the baseline. Even low-risk pixels may experience some change. The full project average provides the base magnitude, which is then risk-weighted in Step H10. This is consistent with the SD VISta formula in Step H10, which shows that risk weighting via w is applied to the average delta, not that the average should be calculated only over high-risk pixels.

The stepwise calculation proceeded as follows. First, delta values from Step H8 were extracted for all project pixels from p equals 1 to 15,636. Second, all delta values were summed. The sum of ΔI at pixel p and time tx equals -48,315.2 percentage points. The negative value indicates overall decline across the program as expected given the historical trends. Third, the total project pixels were counted: P_proj equals 15,636 pixels. Fourth, the mean was calculated: ΔĪ at time tx equals -48,315.2 divided by 15,636, which equals -3.092076878896544 percentage points. Fifth, rounding for reporting: ΔĪ approximately equals -3.09 percentage points.

The result is that the average delta equals -3.09 percentage points over the 5-year monitoring period. The interpretation is that on average across the entire project extent, tree cover is expected to decline by 3.09 percentage points over the 5-year monitoring period based on continuation of historical trends from the 10 eyars previous to the start of the program. This represents an unweighted average annual rate of -3.09 divided by 5 equals -0.62% per year. This baseline value will be risk-weighted in Step H10 to account for the proportion of high-risk area which was calculated as 64.2% in Step H6.

Step H10. Standardize the area-weighted average absolute change

Standardize the area-weighted, average absolute change in the structure indicator using the high-risk proportion of the project Extent and the indicator’s reference value (Sections 7.3.13–7.3.24) as follows:

\[ \Delta C_{\text{pre}} = \frac{\overline{\Delta I}_{t_x} \times w}{R_v} \]

Where:

  • \(\Delta C_{\text{pre}}\) = Area-weighted, standardized predicted change in condition

  • \(\overline{\Delta I_{t_x}}\) = Average absolute change in structure indicator for the set of high-risk pixels in the project extent

  • \(w\) = Proportion of the project extent assessed as high-risk (Step H6)

  • \(R_v\) = Reference value for the structure indicator

The result is that the standardized predicted change equals ΔC_pre equals -0.0345, or expressed as percentage, -3.45% over the 5-year monitoring period. This represents the risk-weighted, normalized expected decline in ecosystem condition under the business-as-usual scenario.

The interpretation of risk weighting effect is important. The risk weighting parameter w equals 0.642 scales the baseline according to the spatial distribution of risk. This calculation follows SD VISta Section A1.2 Step H10 exactly as specified. The standardized change accounts for two key factors: spatial risk distribution via weighted proportion w, and baseline conditions via reference value normalization R_v. Both components reflect actual program characteristics rather than arbitrary assumptions, ensuring the baseline is specific to this program’s unique context and threat profile.

Step H11. Use annual predicted rate of change to estimate crediting baseline

The Crediting baseline (Bt) was calculated as the ratio between the delta (Change in condition) and time (number of years in the monitoring period), as follows:

\[ \hat{\beta}_{\text{pre}} = \frac{\Delta C_{\text{pre}}}{t_x} \]

Where:

  • \(\hat{\beta}_{\text{pre}}\) = Annual estimated rate of change (i.e., estimated crediting baseline)

  • \(\Delta C_{\text{pre}}\) = Area-weighted, standardized predicted change in condition

  • \(t_x\) = Number of years in the monitoring period

The components are: ΔC_pre equals -0.0345, representing fractional change over the monitoring period, and t_x equals 5 years, the monitoring period duration. The calculation is: β-hat_pre equals ΔC_pre divided by t_x, which equals -0.0345 divided by 5, which equals -0.0069 per year. The result is that the annual crediting baseline equals -0.0069 per year, or expressed as percentage, -0.69% per year. This represents the expected annual rate of decline in tree cover under the business-as-usual scenario. This calculation follows SD VISta Section A1.2 Step H11 exactly as specified. The annualized baseline represents the expected annual rate of habitat conversion, is based on historical conversion rates embedded via regression, accounts for spatial risk distribution via weighted proportion, is normalized to baseline conditions via reference value, and serves as the reference for credit calculation at verification. This baseline of -0.69% per year will be compared to actual observed changes during verification events, with credits awarded for performance that exceeds (is better than) this baseline scenario.
